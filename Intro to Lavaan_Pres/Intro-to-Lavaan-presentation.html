O pacote lavaan para modelação de equações estruturais no R

  • Software de distribuição livre, não comercial
  • Desenvolvido por Yves Rosseel, Department of Data Analysis, Ghent University
  • Suporta um extenso número de procedimentos utilizados em SEM:
    • Análise Fatorial Confirmatória - CFA()
    • Modelação de Equações Estruturais - SEM()
    • Modelação geral de estruturas de médias e covariâncias - lavaan()
    • Permite modelar variáveis contínuas, dicotómicas ou ordinais

Especificação de modelos

Diagramas de pistas

  • A especificação de modelos é frequentemente feita com recurso a diagramas de pistas
    • Os diagramas podem incluir 5 tipos de formas:

Retângulos (ou quadrados) -> variáveis observadas (VO)
Círculos -> variáveis latentes (VL)
Setas unidirecionais -> coeficientes de regressão
Setas bi-direcionais -> variâncias ou covariâncias
Triângulos -> constantes (1) para médias ou intercepts

Diagramas de pistas

Correspondência direta com a equação de regressão:

\[y_{i}= \alpha + \gamma_{1}x_{1i} + \zeta_{i} ~~~~~~~~~~~~~~~~~~~~ \zeta_{i} \sim N(0,\Psi)\] \(\alpha\) (alfa) = intercept de y
\(\gamma\) (gama) = coeficiente de regressão
\(\zeta\) (zeta) = resíduo

Diagramas de pistas

Dependendo das fontes, poderão encontrar-se algumas variações deste tipo de diagramas

  • Existem aplicações que permitem especificar modelos com recurso a diagramas:
    • Amos
    • Onyx

Fórmula do modelo

  • Linguagem que descrever o modelo (ver parte 1)
    • Exemplo: y ~ x + z
  • Um modelo no lavaan é constituido por um sistema de fórmulas de equações
meu_model <- "y ~ x + m
              m ~ x"
  • Cada linha de um modelo é uma fórmula com 3 partes:
    • “left-hand side” (lhs);
    • “operator” (op);
    • “right-hand side” (rhs).

Exemplo: y (lhs) ~ (op) x + z (rhs)

Operadores e modificadores

Tipo Operador Exemplo
variável latente =~ x =~ x1 + x2 (x é medido por x1 e x2)
regressão ~ y ~ x (regressão de y em x)
(co)variância ~~ x ~~ y (correlação entre x e y)
intercept ~1 x~1 (representa o intercept de x)
escala * x =~ 1x1 + 1x2 (usado para fixar parâmetros)
definir par. := y ~ cx + bm; m ~ ax; ind := ac (usado para especificar parâmetros)
igualar par. == x =~ ax1 + bx2; a==b (usado para impor igualdade entre parâmetros

Operadores e modificadores

  • Elementos no rhs podem ser precedidos de modificadores
    • Servem geralmente para libertar ou fixar parâmetros
meu_modelo <- "
          # Variáveis latentes
          x =~ NA*x1 + x2 + x3
          y =~ NA*y1 + y2 + y3
          z =~ NA*z1 + z2 + z3
          # correlações entre fatores
          x ~~ 1*x
          y ~~ 1*y
          z ~~ 1*z
          "

Operadores e modificadores

: - … ou para impor condições

meu_modelo <- "
          x =~ L1.1*x1 + L1.2*x2 + L1.3*x3
          y =~ L2.1*y1 + L2.2*y2 + L2.3*y3
          z =~ L3.1*z1 + L3.2*z2 + L3.3*z3
          # Condições
          L1.1 == L2.1
          L1.2 == L2.2
          L1.3 == L2.3
          L3.1 == (L3.2 + L3.3)^2
          "

A função lavaan()

Vários argumentos da função lavaan() podem ser utilizados para automaticamente fixar parâmetros e identificar os modelos

meu_modelo <- "
          # Variáveis latentes
          x =~ L1.1*x1 + L1.2*x2 + L1.3*x3
          y =~ L2.1*y1 + L2.2*y2 + L2.3*y3
          z =~ L3.1*z1 + L3.2*z2 + L3.3*z3
          # Correlações entre fatores
          x ~~ y + z
          y ~~ z
          "
fit <- lavaan(meu_modelo, data = meus_dados, # nome do modelo e dados
              auto.fix.first = TRUE,  # fixa a 1.0 o primeiro loading de cada latente
              auto.var = TRUE) # estima variâncias e residuais
?lavOptions # ver a lista de argumentos

As funções cfa() e sem()

  • Funções de segunda ordem para ajustar modelos: cfa() e sem()

  • cfa() e sem() definem um conjunto de opções por defeito, nomeadamente:
    • auto.var = TRUE (estima variâncias);
    • auto.cov.lv.x = TRUE (estima covariâncias entre variáveis exógenas);
    • auto.fix.first = TRUE (fixa primeiro loading a 1.0).

Regressão linear simples

Efeito da prática de exercício na incidência de problemas de saúde

  • A fórmula do modelo (GLM) é já conhecida:

\[\hat{Y_{i}} = \beta_0 + \beta_1X_{i1} + \beta_2X_{i2} + ... + \beta_pX_{ip}\]

\[{Y_{i}} = \beta_0 + \beta_1X_{i1} + \beta_2X_{i2} + ... + \beta_pX_{ip} + \varepsilon_{i}\]

Logo… \[\varepsilon_{i} = {Y_{i}} - \hat{Y_{i}}\]

Regressão linear simples

A fórmula do GLM aplicada às nossas variáveis toma a seguinte forma:

\[\hat{Iillness_{i}} = \beta_0 + \beta_1Exercise_{i}\]
Onde:

\(\hat{Iillness_{i}}\) = score de illness estimado
\(Exercise_{i}\) = score de exercise
\(\beta_0\) = valor de illness quando exercise assume o valor zero
\(\beta_1\) = coeficiente de regressão (i.e., a mudança esperada em illness em função de exercise)

Estimação OLS do modelo

  • Como já vimos, um modelo de regressão pode ser testado no R com o método OLS usando a função lm()
lm_fit.1 <- lm(illness ~ exercise, data = DF_roth)
summary(lm_fit.1)$coefficients # mostrar parametros

## (Intercept) 719.77420391  3.7923150 189.798105 0.0000000
## exercise     -0.07516391  0.0486226  -1.545864 0.1229901

Modelo de regressão múltipla: efeito de exercise, hardy, fitness e stress em illness

  • Expandimos agora o modelo anterior para incluir, também “hardy”, “fitness” e “stress” como preditores de “illness.”

\[\hat{Iillness_{i}} = \beta_0 + \beta_1Exercise_{i} + \beta_2hardy_{i} + \beta_3fitness_{i} + \beta_4stress_{i}\]
Onde:
\({Illness_{i}}\) = score de illness
\(Exercise_{i}\) = score de exercise
\(hardy_{i}\) = score de hardy
\(fitness_{i}\) = score de fitness
\(stress_{i}\) = score de stress
\(\beta_0\) = valor de illness quando exercise assume o valor zero
\(\beta_1\), \(\beta_2\), \(\beta_3\) & \(\beta_4\) = coeficientes de regressão para X1, X2, X3 e X4

O objeto “model.1” define a fórmula do modelo

model.1 <- "
illness ~ exercise" # illness em função de exercise

Estimação OLS do modelo

lm_fit.2 <- lm(formula = illness ~ exercise + hardy + fitness + stress, data = DF_roth)

  • Os parâmetros \(\beta_1\), \(\beta_2\), \(\beta_3\) & \(\beta_4\) são interpretados respetivamente como os efeitos de exercise, hardy, fitness e stress sobre illness, mantendo constantes os restantes preditores. \(\beta_0\) representa o valor esperado de illness quando todos os preditores assumem o valor de zero.

Modelos de Mediação

De seguida reproduzimos o modelo de mediação apresentado por Rex Kline em Principles and Practice of SEM (3ª ed.), figure 8.1.

No modelo de regressão múltipla anterior, fitness e stress estavam especificados unicamente como variáveis dependentes

No modelo seguinte fitness e stress assumem o papel de variáveis mediadoras

  • São especificados os seguintes efeitos:
    • os efeitos diretos de hardy e exercise nas variáveis Stress e fitness, respectivamente;
    • os efeitos indiretos de hardy e exercise sobre illness via stress e fitness, respetivamente.

Este modelo acrescenta: (1) efeitos diretos de exercise e hardy sobre illness; (2) covariância entre fitness e stress

Modelos de Mediação (Re-especificação)

O modelo anterior traduz-se na seguinte formula no lavaan:

model.4 <- "
# regressões
fitness ~ a1*exercise
stress ~ a2*hardy
illness ~ b1*fitness + b2*stress + c1*exercise + c2*hardy

# covariações
fitness ~~ stress

ind1 := a1*b1
total1 := c1 + (a1*b1)

ind2 := a2*b2
total2 := c2 + (a2*b2)
"
  • Podemos definir os efeitos totais e indiretos recorrendo ao operador :=

Modelos de Mediação (Re-especificação)

Para estimar os intervalos de confiança dos efeitos indiretos utilizamos o bootstraping apenas com 100 amostras.
- Na prática este número deve ser bastante maior, sendo que o valor definido por defeito pelo lavaan é de 5000 amostras.

fit.4 <- sem(model.4, data=DF_roth, meanstructure=T, se = "bootstrap", bootstrap = 100)
summary(fit.4, fit.measures = TRUE, standardized = TRUE, rsquare = TRUE)
